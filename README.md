# E-commerce Sales Analysis SQL Project

## Project Overview
This project analyzes an e-commerce sales database using SQL. The goal is to extract business insights about revenue, customers, products, categories, payment methods, and order performance.

## Dataset
The project contains four related tables:

1. `customers`
2. `products`
3. `orders`
4. `order_items`

The database includes:
- 250 customers
- 80 products
- 900 orders
- 1588 order items

## Tools Used
- SQL
- SQLite
- DB Browser for SQLite / SQLiteStudio
- CSV files

## Main Business Questions
- What is the total revenue?
- Which months generated the most revenue?
- Which products and categories performed best?
- Which cities and customer segments generated the most sales?
- Who are the highest-spending customers?
- What is the distribution of order statuses?
- Which payment methods are most used?

## Key Result Snapshot
- Total delivered revenue: $393,303.55
- Delivered orders: 863
- Active customers with delivered orders: 241
- Products sold: 2414

## Files
- `ecommerce_sales.sqlite`: ready-to-use SQLite database
- `schema.sql`: database table structure
- `analysis_queries.sql`: SQL analysis queries
- `customers.csv`
- `products.csv`
- `orders.csv`
- `order_items.csv`
- `resume_bullets.txt`

## How to Use
1. Open `ecommerce_sales.sqlite` in DB Browser for SQLite or SQLiteStudio.
2. Open the file `analysis_queries.sql`.
3. Run each query one by one.
4. Export query results or screenshots for GitHub documentation.
